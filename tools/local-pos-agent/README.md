# PayMyDine Local POS Agent

PayMyDine local POS agent runs on the **Windows POS machine** and supports two production paths:

1. **Queue polling path** (backend enqueue -> agent pull -> local execute -> ack)
2. **Localhost bridge path** (admin UI -> `http://127.0.0.1:17877` -> local execute)

## Requirements

- Node.js 18+
- Network access from POS machine to tenant admin URL

## Environment (`.env` next to `agent.js`)

```env
BACKEND_BASE_URL=https://your-tenant.paymydine.com/admin
POS_AGENT_TOKEN=replace_with_same_token_as_backend
POS_DEVICE_CODE=your_pos_device_code
POS_PAIRING_TOKEN=token_from_admin_device_record
POS_DISPLAY_NAME=Front Counter POS
POLL_INTERVAL_MS=2000

# Localhost bridge settings
LOCAL_API_ENABLED=true
LOCAL_API_HOST=127.0.0.1
LOCAL_API_PORT=17877
DEFAULT_DRAWER_TARGET=
LOCAL_TEST_PRINT_TEXT=PayMyDine printer test
```

## Run locally

```bash
node agent.js
```

## Localhost API (binds only to 127.0.0.1 by default)

- `GET /health`
- `GET /printers` (Windows implementation)
- `POST /open-drawer`
- `POST /test-drawer`
- `POST /test-print`

Example test drawer body:

```json
{
  "target": "192.168.1.50:9100",
  "esc_pos_command": "27,112,0,60,120",
  "printer_name": "EPSON TM-T20"
}
```

## Queue polling behavior

- Pull endpoint: `GET {BACKEND_BASE_URL}/api/pos-agent/commands/pull?device_code=...`
- Ack endpoint: `POST {BACKEND_BASE_URL}/api/pos-agent/commands/{id}/ack`
- Pair endpoint: `POST {BACKEND_BASE_URL}/api/pos-agent/pair` (if pairing token exists)

## Windows connector defaults (generated by admin)

- folder: `C:\ProgramData\PayMyDine\LocalPosAgent`
- log: `C:\ProgramData\PayMyDine\LocalPosAgent\agent.log`
- startup task: `PayMyDineLocalPosAgent`
