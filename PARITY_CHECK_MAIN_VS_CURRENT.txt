SECURITY IMPROVEMENTS
========================================

✅ CRITICAL FIXES:
1. app/main/routes.php /api/v1 group now has DetectTenant middleware
   - OLD: Completely unprotected (App::before bypass)
   - NEW: Tenant-protected with middleware
   - Impact: Closes major data leak vulnerability

2. DetectTenant double-prefix bug fixed
   - OLD: Queries ti_ti_tenants (fails)
   - NEW: Queries ti_tenants (works)
   - Impact: Tenant detection now functional

3. No-subdomain requests now explicitly rejected
   - OLD: Silently falls back to default DB (data leak)
   - NEW: Returns 404 JSON (fail-safe)
   - Impact: Prevents accidental data exposure

4. Dynamic table prefixing restored
   - OLD: Hardcoded ti_* in 18 locations
   - NEW: All use {$p}* variable
   - Impact: Supports configurable prefixes

✅ ENHANCEMENTS:
5. Phase 2 cache isolation
   - Per-tenant cache prefix: laravel:tenant:<slug>
   - Prevents cross-tenant cache pollution

6. Phase 2 session isolation
   - Session bound to tenant_id
   - Cross-tenant reuse triggers invalidation + regeneration
   - Prevents session hijacking across tenants

7. Rate limiting on public writes
   - throttle:30,1 on waiter-call, table-notes, etc.
   - Prevents abuse and DoS attacks
